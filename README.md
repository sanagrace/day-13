# day-13

# What is CI/CD?
CI/CD stands for Continuous Integration and Continuous Deployment/Delivery. It automates software building, testing, and deployment processes.

# Why deployment workflow matters?
A deployment workflow ensures changes are tested and released in a controlled manner. It reduces errors and improves software quality.

# Problems without version control?
Without version control, code changes can be lost or overwritten. Teams may face conflicts and difficulty tracking modifications.

# GitHub + DX + DevOps explanation?
GitHub manages source code, Salesforce DX supports development, and DevOps automates delivery processes. Together they improve collaboration and deployment efficiency.

# Enterprise deployment risks?
Deployment risks include system downtime, data loss, security issues, and failed releases. Proper testing and automation help minimize these risks.

# Reflection
CI/CD, GitHub, and Salesforce DX enable faster and more reliable software delivery. They help organizations maintain quality and support team collaboration.
